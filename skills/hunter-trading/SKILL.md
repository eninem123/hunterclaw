### R01 涨停过滤规则
```
条件: 涨幅 >= 9.8%
动作: 禁止买入（信号队列自动拒绝）
理由: 涨停股流动性极差，次日高开概率低
```

### R03 量比门槛（v3.13温度自适应）
```
条件: 量比 < 动态阈值
  ≥70℃市场: 量比≥1.0（放量市场门槛降低）
  50-70℃市场: 量比≥1.3（正常标准）
  35-50℃市场: 量比≥1.2（缩量市场适当降低）
  <35℃市场: 不操作
动作: 不达标则禁止买入
理由: 量比随市场温度动态调整，高温放量市场门槛放宽
```

### R08 分批止盈（v3.13温度自适应档位）
```
≥70℃市场（放量趋势市）：
  触发点1: 浮盈 +5%  → 卖出1/3
  触发点2: 浮盈 +10% → 卖出1/3
  触发点3: 浮盈 +15% → 卖出剩余

50-70℃市场（正常市）：
  触发点1: 浮盈 +3%  → 卖出1/3
  触发点2: 浮盈 +6%  → 卖出1/3
  触发点3: 浮盈 +10% → 卖出剩余

理由: 分批锁定利润，高温市场可让利润奔跑更久
```

## 操作流程

### 标准分析流程

```
用户请求 → 解析意图 → 获取数据 → 执行分析 → 生成报告 → 计费解锁
     ↓           ↓           ↓           ↓           ↓           ↓
  股票代码    技术分析    行情数据    规则引擎    Markdown    支付宝
  +需求      +情绪判断   +市场数据   +风控评估    报告        收款
```

### 步骤1：接收用户请求

解析用户输入，识别：
- 股票代码（支持代码和名称）
- 分析需求类型（查询/分析/分析参考）
- 是否包含持仓信息

### 步骤2：获取市场数据

调用数据接口获取：
- 实时行情数据（价格、涨跌幅、成交量）
- 市场环境数据（涨跌家数、成交额）
- 情绪指标（恐贪指数、涨停家数）

### 步骤3：执行策略分析

按规则引擎执行多维度分析：

**技术分析（R35/R38规则）**：
- 计算回撤幅度、量比、涨幅
- 判断是否触发低位启动信号
- 分析连板情况和板块联动

**情绪分析（R6/R7规则）**：
- 读取恐贪指数
- 判断情绪周期阶段
- 计算仓位上限

**风控评估（R2/R3规则）**：
- 计算止损位
- 评估主力诱多风险
- 判断赚钱效应

### 步骤4：生成报告

整合分析结果，生成结构化报告：

```markdown
# 猎手分析报告 - {股票名称}

## 一、行情概览
- 当前价：{价格}
- 涨跌幅：{涨跌幅}
- 成交量：{成交量}

## 二、技术分析
- 信号：{ bullish/neutral/bearish }
- 关键价位：支撑位、压力位
- 形态判断：{形态名称}

## 三、情绪判断
- 恐贪指数：{数值}
- 情绪阶段：{阶段名称}
- 仓位参考：{仓位比例}

## 四、风险评估
- 止损价：{价格}
- 风险等级：{高/中/低}
- 诱多风险：{是/否}

## 五、分析参考
- 参考：{买入/持有/卖出}
- 仓位：{仓位参考}
- 理由：{具体理由}

⚠️ 风险提示：本报告仅供模拟交易参考，不构成投资参考。
```

### 步骤5：计费与解锁

1. 根据用户请求的功能类型确定价格
2. 调用支付宝AI收接口发起收款
3. 支付完成后解锁完整报告
4. 未支付用户仅显示摘要

## 部署指南

### 本地部署（方式B - 推荐）

```bash
# 1. 创建Skill目录
mkdir -p ~/.openclaw/skills/hunter-trading

# 2. 复制文件
cp -r ./猎手Skill/* ~/.openclaw/skills/hunter-trading/

# 3. 配置支付
# 编辑 config/payment.yaml 填入支付宝AppID
```

### 配置文件

创建 `~/.openclaw/skills/hunter-trading/config/payment.yaml`：

```yaml
alipay:
  app_id: "2021006149605908"
  price: 0.05
  unit: "次"
  enabled: true

openclaw:
  skill_path: "~/.openclaw/skills/hunter-trading"
  local_mode: true  # 本地模式，无需公网服务器
```
