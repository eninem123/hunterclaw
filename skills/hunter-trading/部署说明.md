# 猎手交易分析Skill - 部署说明

> 版本：v1.0 | 部署到龙虾(OpenClaw)平台

---

## 一、部署方式选择

### 方式A：HTTP API服务（需要公网服务器）

**适用场景**：
- 需要对外提供服务
- 有公网服务器资源
- 需要高并发支持

**部署步骤**：
1. 部署API服务到公网服务器
2. 配置Nginx反向代理
3. 在OpenClaw配置Webhook回调
4. 配置支付宝支付回调

**缺点**：
- 需要公网服务器成本
- 配置复杂
- 维护成本高

---

### 方式B：本地加载（推荐）

**适用场景**：
- 个人/小团队使用
- 无公网服务器
- 快速部署

**优势**：
- 零服务器成本
- 配置简单
- 易于维护

**部署步骤**：见第二章

---

## 二、本地部署（方式B）

### 2.1 前提条件

- 安装了 OpenClaw（龙虾）平台
- 已安装 Node.js ≥ 22（如使用CLI）
- 拥有支付宝账号（用于收款）

### 2.2 安装步骤

#### 步骤1：创建Skill目录

```bash
# 在OpenClaw skills目录下创建猎手交易Skill
mkdir -p ~/.openclaw/skills/hunter-trading
```

#### 步骤2：复制Skill文件

```bash
# 复制所有文件
cp -r ./猎手Skill/* ~/.openclaw/skills/hunter-trading/

# 验证文件
ls -la ~/.openclaw/skills/hunter-trading/
```

#### 步骤3：配置支付

```bash
# 编辑支付配置文件
nano ~/.openclaw/skills/hunter-trading/config/payment.yaml
```

填入你的支付宝AppID：
```yaml
alipay:
  app_id: "2021006149605908"  # 你的AppID
  price: 0.05
  enabled: true
```

#### 步骤4：配置私钥

```bash
# 创建.env文件存储私钥
cat > ~/.openclaw/skills/hunter-trading/.env << 'EOF'
ALIPAY_APP_ID=2021006149605908
ALIPAY_PRIVATE_KEY=你的应用私钥（从SECRET.md获取）
EOF

# 设置文件权限
chmod 600 ~/.openclaw/skills/hunter-trading/.env
```

### 2.3 验证安装

```bash
# 检查OpenClaw是否识别Skill
openclaw skills list

# 应该看到 hunter-trading 在列表中
```

### 2.4 启动使用

在OpenClaw平台中：
1. 打开对话窗口
2. 输入分析请求，如："分析贵州茅台"
3. 系统自动加载猎手交易Skill
4. 完成后显示报告

## 四、支付宝配置

### 4.1 获取AppID

1. 登录支付宝开放平台：https://open.alipay.com/
2. 进入控制台 → 创建应用
3. 获取AppID

### 4.2 配置应用私钥

1. 使用支付宝密钥工具生成RSA2密钥对
2. 将私钥保存到安全位置
3. 在应用配置中上传公钥
4. 将私钥配置到Skill的`.env`文件中

### 4.3 开通AI收功能

1. 在支付宝开放平台找到"AI收"产品
2. 开通服务
3. 配置回调地址（如果使用API模式）

### 4.4 沙箱环境

测试阶段可使用沙箱环境：
1. 访问：https://open.alipay.com/develop/sandbox
2. 获取沙箱AppID和密钥
3. 配置到本地环境

## 六、常见问题

### Q1: 支付无法完成？
A: 检查支付宝AppID和私钥是否正确配置，确认AI收功能已开通。

### Q2: Skill无法加载？
A: 检查文件路径是否正确，确认OpenClaw版本支持自定义Skill。

### Q3: 数据获取失败？
A: 检查网络连接，确认行情数据接口可用。

### Q4: 如何查看调用记录？
A: 配置文件中有日志配置，查看logs目录或控制台输出。

## 七、更新升级

### 手动更新

```bash
# 备份当前版本
cp -r ~/.openclaw/skills/hunter-trading ~/.openclaw/skills/hunter-trading.bak

# 拉取新版本
git pull  # 如果使用git管理

# 或直接复制新文件
cp -r ./猎手Skill/* ~/.openclaw/skills/hunter-trading/
```

### 保留配置

更新时保留自定义配置：
```bash
# 只更新SKILL.md和references
cp ~/.openclaw/skills/hunter-trading/config/payment.yaml /tmp/
cp -r ./猎手Skill/* ~/.openclaw/skills/hunter-trading/
cp /tmp/payment.yaml ~/.openclaw/skills/hunter-trading/config/
```
