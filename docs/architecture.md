# HunterClaw 系统架构

## 1. 系统概述

HunterClaw 是基于 OpenClaw Multi-Agent 框架的 A 股智能交易系统，采用三层 AI Agent 协作架构。

## 3. Agent 职责

### 3.1 Claw（主控Agent）

| 属性 | 说明 |
|------|------|
| 运行时 | OpenClaw main session |
| 角色 | 执行者、协调者 |
| 核心能力 | 代码执行、文件操作、消息推送 |

### 3.2 Hermes（策略Agent）

| 属性 | 说明 |
|------|------|
| 运行时 | Hermès CLI subagent |
| 角色 | 策略顾问、记忆整理 |
| 触发方式 | Cron 15:30 / 22:00 |
| 输出 | 策略优化建议.md |

### 3.3 GBrain（知识库）

| 属性 | 说明 |
|------|------|
| 运行时 | PGLite 本地数据库 |
| 角色 | 长期记忆、语义搜索 |
| 规则 | 先查GBrain再调外部API |

## 4. 数据流

### 4.1 盘中交易流程

```
1. Cron触发（9:30-14:30每30分钟）
       │
       ▼
2. auto_trade.py 执行检测
       │
       ├──→ 检查持仓信号（止损/止盈）
       │
       ├──→ 计算市场温度
       │
       └──→ 判断是否触发熔断
       │
       ▼
3. 生成持仓报告
       │
       ▼
4. 写入 pending-summaries/
       │
       ▼
5. 心跳检测到 → 推送微信
```

### 4.2 收盘复盘流程

```
1. Cron触发（15:30）
       │
       ▼
2. strategy_analysis.sh 执行
       │
       ▼
3. Hermes 生成策略分析
       │
       ├──→ 读取持仓盈亏
       │
       ├──→ 读取今日推演
       │
       └──→ 对比预测vs实际
       │
       ▼
4. 输出策略优化建议.md
       │
       ▼
5. 推送微信
```

## 5. 安全机制

### 5.1 熔断机制

```
总回撤 ≥ 3% → 触发熔断
       │
       ├──→ 禁止新买入
       └──→ 必要时清仓
```

### 5.2 止损止盈

```
止损：-5%（绝对红线，触发必走）
止盈：+10%（达标分批走）
```

### 5.3 仓位限制

```
单日买入：≤ 3次
单只仓位：≤ 30%
```

## 6. 技术栈

| 组件 | 技术 |
|------|------|
| 主框架 | OpenClaw 2026.4.x |
| 策略Agent | Hermes CLI |
| 知识库 | GBrain v0.7.0 + PGLite |
| 数据获取 | 腾讯行情API / 同花顺 |
| 定时调度 | Linux Cron |
| 推送渠道 | WeChat (OpenClaw Plugin) |
