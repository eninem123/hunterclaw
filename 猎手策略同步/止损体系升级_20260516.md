# 猎手三级止损体系
> 版本：v1.0 | 生效日期：2026-05-16 | 基于：5/15亏损-1619元教训

---

## 一、设计背景

### 1.1 问题诊断
5/15单日亏损-1619元，止损太频繁但不精准：
- 止损触发后才发现卖出时机已过
- 缺乏"提前预警"机制，往往跌到位才动
- 止损执行不果断，抱侥幸心理
- 三级止损没有形成闭环，各自为战

### 1.2 解决思路
借鉴专业止损体系（固定止损+技术止损+ATR止损），结合猎手已有规则（FM-031/FM-034），构建**三级止损闭环**：
- **一级**：预警止损（FM-034）→ 提前减仓
- **二级**：硬止损（-5%）→ 截断亏损
- **三级**：极端止损（FM-031）→ 识别主力出货

---

## 二、三级止损体系详解

### 2.1 一级止损：预警止损（FM-034触发）

**定位**：提前0.8%预警，比-5%止损更早行动

**触发条件（优化参数）**：
| 条件 | 参数 | 说明 |
|------|------|------|
| 跌幅 | >3% | 单日跌幅超过3% |
| 量比 | >2.5 | 成交量是均量的2.5倍以上 |
| 位置 | 高位/破位 | 股价处于相对高位或跌破支撑 |

**操作动作**：
```
触发FM-034 → 立即减仓50% → 剩余仓位设置追踪止损
```

**回测数据（31样本）**：
- 命中率：86.2%
- 提前预警：100%案例比-5%止损早触发
- 平均提前幅度：-0.8%
- 5日继续下跌概率：86.7%

**执行时机**：
- 次日开盘执行减仓（避免T+1限制）
- 设置止损条件单：现价-5%自动清仓剩余

---

### 2.2 二级止损：硬止损（-5%截断）

**定位**：核心铁律，不可放宽，不可动摇

**触发条件**：
| 条件 | 参数 | 说明 |
|------|------|------|
| 累计跌幅 | ≥5% | 从买入成本计算 |

**操作动作**：
```
触发-5%止损 → 立即清仓 → 记录亏损 → 复盘原因
```

**执行要点**：
- 盘中跌破-5%：14:30前无条件清仓
- 收盘跌破-5%：次日开盘清仓
- **禁止**：跌到-4.9%侥幸等待反弹
- **禁止**：补仓摊薄成本

**配套规则**：
- 单日最大止损次数：2次（防止频繁割肉）
- 单只最大亏损：总仓位的-1.5%（≈-5%×30%仓位）

---

### 2.3 三级止损：极端止损（FM-031触发）

**定位**：识别主力高位出货，立即清仓不犹豫

**触发条件（优化参数）**：
| 条件 | 参数 | 说明 |
|------|------|------|
| 量比 | >2.5 | 成交量异常放大 |
| 涨幅 | <1.5% | 放量但不涨 |
| 换手率 | >12%持续2日 | 主力在派发 |
| 位置 | 高位 | 涨幅>20%后的高位 |

**操作动作**：
```
触发FM-031 → 立即清仓（不犹豫）→ 不等反弹
```

**回测数据（14样本）**：
- 命中率：100%
- 平均跌幅：7.8%
- 最大跌幅：11.86%（鑫宏业）

**执行要点**：
- 这是"主力已经在跑"的信号
- **禁止**：想"等反弹再卖"
- **禁止**：觉得"已经跌这么多了"
- 主力跑的比你快，散户必须更快

## 三、配套止损机制

### 3.1 追踪止损（让利润奔跑）

**适用场景**：持仓盈利后，锁定已有利润

**操作规则**：
| 盈利状态 | 追踪止损位置 |
|----------|--------------|
| +3%~+5% | 成本价（保本） |
| +5%~+8% | +3%处 |
| +8%~+10% | +5%处 |
| >+10% | +8%处（分批止盈） |

**执行原则**：
- 止损位只上移，不下移
- 以收盘价为准（避开盘中假跌破）

### 3.2 ATR动态止损（适应波动）

**已有规则（MEMORY.md）**：
| ATR比例 | 市场状态 | 仓位调整 | 止损系数 |
|---------|----------|----------|----------|
| <0.7 | 低波动蓄势 | +20% | 2.0×ATR |
| 0.7-1.3 | 正常波动 | 标准 | 1.5×ATR |
| 1.3-2.0 | 高波动预警 | -30% | 1.0×ATR |
| >2.0 | 极高位 | -50% | 0.8×ATR |

**应用**：
- 高波动股（周期股、科技股）：止损放宽至2×ATR
- 低波动股（银行、电力）：止损收紧至1×ATR

### 3.3 时间止损（对抗拖延）

**触发条件**：
| 持仓时长 | 条件 | 操作 |
|----------|------|------|
| 3个交易日 | 未盈利+未触发止损 | 减仓50% |
| 5个交易日 | 未盈利+未触发止损 | 清仓 |
| 买入逻辑 | 已证伪 | 立即清仓 |

**逻辑止损触发**：
- 基本面恶化（业绩暴雷、政策打压）
- 题材证伪（利好未兑现）
- 行业逻辑逆转

## 四、止损执行检查清单

### 4.1 触发前（每日检查）
```
□ 检查持仓现价距止损位距离
□ 检查个股ATR波动率
□ 检查市场整体灯色（黄灯/红灯升级止损）
□ 检查是否有FM-034/FM-031预警信号
```

### 4.2 触发时（立即执行）
```
□ 确认触发哪一级止损
□ 确认操作动作（减仓/清仓）
□ 确认执行时间（盘中/次日开盘）
□ 确认T+1限制（今日买的不能卖）
□ 设置新的止损条件单（剩余仓位）
```

### 4.3 触发后（复盘归档）
```
□ 记录亏损金额
□ 分析触发原因
□ 复盘买入逻辑是否错误
□ 更新规则库（如发现新模式）
□ 记录教训到 ./猎手模拟交易/止损复盘_YYYYMMDD.md
```

## 六、与猎手现有规则的整合

### 6.1 规则映射

| 新三级止损 | 对应现有规则 | 整合方式 |
|------------|--------------|----------|
| 一级预警止损 | FM-034放量下跌止损 | 直接采用，优化参数 |
| 二级硬止损 | -5%铁律 | 保持不变 |
| 三级极端止损 | FM-031高位出货 | 直接采用，优化参数 |
| 追踪止损 | ZS-01止盈分批 | 升级为三级联动 |
| ATR动态止损 | MEMORY.md已有 | 保持不变 |

### 6.2 冲突解决

| 冲突场景 | 解决规则 |
|----------|----------|
| FM-031与FM-034同时触发 | 先执行FM-031（三级优先） |
| -5%与FM-034冲突 | FM-034先减仓，再设-5%止损剩余 |
| 追踪止损vs硬止损 | 硬止损永远优先（亏损优先截断） |
