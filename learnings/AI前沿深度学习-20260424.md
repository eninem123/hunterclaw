# AI前沿深度学习内容 (2026-04-24)

## 1. GenericAgent - 自进化Agent框架

**核心理念**：不预设技能，靠使用进化

**架构亮点**：
- ~3,300行核心代码（对比Claude Code的53万行）
- 7个原子工具：code_run、file_read、file_write、file_patch、web_scan、web_execute_js、ask_user
- 92行Agent Loop - 极简主循环
- 四层记忆机制：工作区检查点 + 长期记忆更新

**自进化机制**：
新任务 -> 自主探索 -> 固化执行路径为Skill -> 写入记忆层 -> 下次直接调用

**关键数字**：
- Token消耗仅为Claude Code的1/6
- 整个GitHub仓库从git init到每条commit都是Agent自主完成

**与猎手系统结合**：
- 可借鉴执行路径固化机制：每次成功交易策略自动保存为可复用规则
- 7个原子工具的设计思路可用于猎手工具集简化

## 2. browser-use - AI浏览器自动化架构

**架构组成**：
- Agent Service (协调者)
- Message Manager (消息管理)
- Controller (动作执行) + Action Registry
- Browser Service (浏览器控制) - Playwright + CDP
- DOM Service (DOM处理) - buildDomTree.js

**执行流程**：
用户任务 -> LLM规划 -> 获取浏览器状态 -> DOM提取 -> LLM决策 -> 执行动作 -> 循环直到完成

**关键特性**：
- 支持保留登录态连接已有Chrome
- Vision + HTML Extraction双重感知
- 自动绕过验证码
- 元素追踪确保重复运行一致性

**与猎手系统结合**：
- 可用于自动登录券商APP查看持仓
- 自动抓取实时行情数据
- 配合MCP协议实现标准化调用

## 3. MCP协议 - Agent与浏览器的标准接口

**MCP (Model Context Protocol)**：AI客户端与工具之间的标准化交互协议

**核心价值**：
- 一次开发，多客户端适配（Claude Desktop、Cursor、Copilot）
- 统一接口：execute_task、get_status、stop_task
- 协议隔离：浏览器扩展不感知MCP细节

**MCP-B架构**：
Tab MCP Server (运行在网页内) -> Chrome Extension (MCP Hub) -> Native Bridge (原生消息隧道) -> AI Assistant

**10,000x性能提升**：
- 传统：10-20秒/任务，$4-5 API成本
- MCP-B：毫秒级直接API调用

**实际应用**：
- 只需~50行代码，网站即可AI-ready
- 使用浏览器现有认证，无需API Key管理
