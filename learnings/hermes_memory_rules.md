## 硬限制规则（绝对不可超）

| 文件 | 硬限制 | 说明 |
|------|--------|------|
| MEMORY.md | 2,200 字符 | 超出自动压缩 |
| USER.md | 1,375 字符 | 超出自动压缩 |
| SOUL.md | 无硬限制 | 但需精简 |

**触发压缩时机**：
- 写入后立即检查
- 目标 Agent 读取时检查
- 发现超标立即压缩

## 记忆写入规则

### 1. 固定规则 → SOUL.md（不占 MEMORY 额度）
- 身份定位（我是谁、做什么）
- 核心价值观（什么是好的、什么不能做）
- 能力边界（我能/不能做什么）

### 2. 非实时写入 → MEMORY.md（需明确触发）
**触发条件**：
- 用户明确说记住xxx
- 用户表现出偏好（用词、风格、习惯）
- 环境事实（时间、地点、项目状态）
- 错误纠正（用户纠正我的错误）
- 里程碑事件（重大成功/失败）

### Skill = 标准化 SOP
每个 Skill 必须包含：
1. **触发条件**：什么时候用
2. **操作步骤**：怎么做
3. **注意事项**：坑在哪里
4. **验证方法**：怎么确认成功

## 多 Agent 协作规则

### 父 Agent → 子 Agent
- 传递完整 context（历史记录）
- 不继承父 Agent 历史记忆
- 独立思考、独立输出

### 子 Agent 并发限制
- 同时活跃 ≤ 3 个
- 超过排队等待

---

## MEMORY.md 精简原则

### 合并同类项
- 多个相似条目 → 合并为一条带示例
- 过期信息 → 删除或归档

### 删除原则
- 已过期的任务/项目
- 已解决的错误
- 重复的规则

### 压缩算法
1. 提取关键信息（去修饰词）
2. 用缩写/代码替代长描述
3. 表格化（节省空间）

## 猎手系统专属记忆规则

### 交易规则（永久记忆）
- 止损 -5%（红线）
- 止盈 +8%
- 单日买入 ≤ 3次
- 单只持仓 ≤ 30%

### 执行纪律（永久记忆）
- 不追高买入
- 信号5分钟未回踩取消
- T+1 规则严格执行
